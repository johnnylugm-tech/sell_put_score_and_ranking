### 8 PoP × ROCC 複合效率分(9 分)- ✅ 完全實作

**代碼實作:**

```python
# Step 1:ROCC 基礎計算
ROCC_raw = (Bid / (Price × 0.20)) × (365 / DTE) × 100

# Step 2:DTE 效率係數
if 30 <= DTE <= 45: dte_factor = 1.10   # Theta 加速期,+10%
elif 20 <= DTE <= 29: dte_factor = 1.00  # 標準(含 DTE=20)
elif 45 < DTE <= 60: dte_factor = 0.95  # 略低效率
else: dte_factor = 0.85                   # DTE < 20 或 > 60

ROCC_adj = ROCC_raw × dte_factor

# Step 3:PoP(獲利機率)
PoP = 1 - abs(Delta)

# Step 4:複合效率分
Efficiency = ROCC_adj × PoP
```

**評分規則:**

| Efficiency Score | 分數 |
|-----------------|------|
| > 400 | 9 |
| 250 - 400 | 7 |
| 150 - 250 | 5 |
| 80 - 150 | 3 |
| < 80 | 1(含所有有報價的合約) |

## 七(新增)、Tier 機制

### 數據來源分層(tier)

`tier` 表示期權 IV 數據的可靠性和來源層級:

| Tier | 等級 | 數據來源 | 可靠性 | 觸發條件 |
|------|------|---------|--------|---------|
| **t1** | 最高 | 真實市場 bid/ask 報價 | ⭐⭐⭐ | 期權鏈 API 返回真實 bid/ask(Spread < 10%)|
| **t2** | 中 | lastPrice 透過 BSM 回算 IV | ⭐⭐ | 無 bid/ask 但有 lastPrice |
| **t3** | 最低 | HV × 1.3 估算 | ⭐ | API 404 / 數據不可得 |

### Tier 顯示規則

`tier` 資訊記錄在 `OptionData.tier` 欄位(`core.py`),但**僅 t2/t3 在 ⚠️ 欄顯示標記**:

```python
tier_mark = "" if tier in ("real", "t1") else f"⚠️{tier}"
```

- **t1**:不顯示標記(✅ 真實報價,正常)
- **t2**:顯示 `⚠️t2`
- **t3**:顯示 `⚠️t3`

### ⚠️ 欄 Legend 說明

```
📌 ⚠️ = 有警告(過熱/財報/IV低估等);無警告通常代表 tier=t1(真實報價)
```

## 七(新版)、希臘字母(Greeks)與Strike偏離

### Gamma(新增)

 - 衡量 Delta 對標的價格變化的加速程度。

| Gamma 範圍 | 意義 |
|-----------|------|
| > 0.003 | ITM/ATM,Delta 極不稳定,適合 Short Gamma 策略 |
| 0.001-0.003 | 適中,需關注 |
| < 0.001 | OTM,Delta 相對穩定 |

### 希臘字母顯示

`tier` 機制已擴展,`OptionData` 現在包含完整的 Black-Scholes Greeks:

```python
@dataclass
class OptionData:
    delta: float          # 標的價格+$1 時期權價格的變化量
    theta: float = 0.0    # 每日時間價值衰減($)
    vega: float = 0.0     # IV+1% 時期權價格的變化量
    strike_deviation: float = 0.0  # (實際履約價 - 8%OTM目標) / 8%OTM目標
```

### Spread%(買賣價差)

`spread` = `(Ask - Bid) / MidPrice × 100%`,代表進出场摩擦成本:

| Spread | 意義 |
|--------|------|
| < 2% | 流動性良好 |
| 2-5% | 尚可 |
| 5-10% | 成本較高 |
| > 10% | 流動性風險高 |

### Strike 偏離警告

| 偏離值 | 警告 | 意義 |
|--------|------|------|
| > +2% | `ITM` | 履約價高於現價,Delta>0.55,風險較高 |
| < -5% | `深OTM` | 履約價遠低於現價,權利金稀薄 |
| 0 ~ -5% | 正常 | 在 8% OTM 目標附近 |

報告主表格新增 `Delta` 欄(位於 RSI 和 距低%之間)。

## 七、年化收益率與時機判定(v5.0_REVISED v3 更新)

### 年化收益率(年化%)

```python
# P0修正4:年化% DTE口徑與表格DTE一致(均用 days_to_earnings)
stock_dte = days_to_earnings if 0 < days_to_earnings < 999 else option_dte
annual_return = IV × 0.05 × √(stock_dte / 365) × 100
# P1修正4:扣除 Spread 摩擦成本,下限 50%(避免 spread=100% 時 annual_return 歸零)
spread_factor = max(1 - spread / 100, 0.5)  # 至少保留 50%
annual_return = annual_return × spread_factor
```

- **DTE 來源**:優先使用 `days_to_earnings`(距財報天數);無財報時 fallback 到 `option.dte`(期權實際到期日)
- **顯示格式**:`162(16-24)` = 理論值(合理區間 Min-Max)
- **合理區間**:理論值 × 10-15%(實際權利金約為理論最大值的 10-15%)
- **小於 1%**:顯示為 `<1`(例如 `1(<1)`)
- **用於**:`report_formatter.py` 主表格「年化%」欄位

### 時機判定(時機)

| 條件 | 結果 |
|------|------|
| DTE < 14 AND RSI > 60 | 短線 |
| 14 ≤ DTE ≤ 45 | 波段 |
| DTE > 45 | 長期 |

用於 `report_formatter.py` 主表格「時機」欄位。
