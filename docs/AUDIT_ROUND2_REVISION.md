## Bug 2 修正說明

### 前版澄清的錯誤

前版說「TSM s7=0 是正確結果，代碼無需修復」——這是**錯誤的**。

`s7=0` 在模型中代表「最高事件風險」，等同於「財報明天就到」。但 TSM 財報剛過，下次約 90 天後，實際事件風險極低。`s7=0` 在這裡是**錯誤懲罰**。

### 已修復內容

**core.py**：
```python
# 新增
elif days_to_earnings < 0: scores['s7'] = 5  # 已過財報：無近期事件風險 → s7=5
```

**doc §7 表格**：
```
# 舊（錯誤）
| 已過財報（個別股票）| 0（無未來事件風險，`days<0` → `else: 0`）|

# 新（正確）
| 已過財報（個別股票）| 5（下次財報約90天後，與 >30天 同等）|
```

**doc §5 標題**：
```
# 舊
代碼實作(無持倉追蹤,無板塊集中度懲罰):

# 新
代碼實作（無外部持倉追蹤，但板塊集中度懲罰使用觀察名單本身）：
```

**doc §9 對比表**：
```

| 板塊集中度懲罰 | 需要持倉追蹤 | ❌ 未實作 |

# 新
| 板塊集中度懲罰 | 直接統計觀察名單中各板塊檔數 | ✅ 已實作 |
```
