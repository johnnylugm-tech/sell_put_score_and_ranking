### ② IV Rank（IVR）— v5.0 核心升級

**研究依據：** Tastytrade IVR/IVP 研究、CBOE PutWrite Index 方法論

**v4.0 問題：** 原始 IV/HV 比無法跨股票公平比較（半導體 IV 天生高於公用事業），導致高 IV 板塊系統性獲得加分，而實際溢價是否豐厚取決於歷史分位而非絕對數值。

**v5.0 計算（雙指標，IVR 為主）：**

```python
# IV Rank（IVR）：今日 IV 在過去 52 週的相對位置
IVR = (IV_today - IV_52w_low) / (IV_52w_high - IV_52w_low) × 100

# IV Percentile（IVP）：過去 252 交易日中，IV 低於今日的比例（輔助展示）
IVP = len([d for d in past_252_days if IV[d] < IV_today]) / 252 × 100

# 原始 IV/HV 比（保留在原始數據頁，不直接計分）
iv_hv_ratio = IV_atm / HV_30day
```

**新評分規則（上限仍 18 分，但改用 IVR）：**

| IVR | 分數 | 解讀 |
|-----|------|------|
| ≥ 80% | 18 | 歷史高位，Put 溢價極豐厚 |
| 60 – 79% | 15 | 高位，溢價豐厚 |
| 40 – 59% | 12 | 中上，溢價適中 |
| 20 – 39% | 9 | 中下，溢價有限 |

# Step 1：計算個股對 SPY 的 90 天滾動 Beta
beta = cov(stock_returns_90d, SPY_returns_90d) / var(SPY_returns_90d)

### ⑧ PoP × ROCC 複合效率分（9 分）— v5.0 核心升級

**研究依據：** Alpaca Wheel GitHub 評分公式 `Score = 年化回報 × P(不被行權)`、Tastytrade 20-Delta 最優研究、Bondarenko (2024) DTE 效率分析

**v4.0 問題：** ROCC 只評估年化回報率，忽略獲利機率。ROCC=500% 但 Delta=0.45 的 Put 遠比 ROCC=180% 但 Delta=0.20 的 Put 風險高，但在 v4.0 中前者反而得 9 分，後者只得 3 分。

**v5.0 計算：**

```python
# Step 1：ROCC 基礎計算
DTE = actual_dte  # 從期權鏈取得
ROCC_raw = (Bid / (Price × 0.20)) × (365 / DTE) × 100  # 年化

# Step 2：DTE 效率係數（基於 Bondarenko 2024 WPUT 研究）
# 30-45 DTE 為 Theta 衰減最優窗口
if 30 <= DTE <= 45:    dte_factor = 1.10   # Theta 加速期，+10% 獎勵
elif 21 <= DTE < 30:   dte_factor = 1.00   # 標準
elif 45 < DTE <= 60:   dte_factor = 0.95   # 略低效率
else:                   dte_factor = 0.85   # Gamma 風險溢出（DTE < 21 或 > 60）

ROCC_adj = ROCC_raw × dte_factor

# Step 3：獲利機率（PoP）
# 使用目標 Delta 的補數（Black-Scholes 近似值）
PoP = 1 - abs(target_delta)
# 例：Delta = 0.20 → PoP = 0.80（80% 機率不被行權）

# Step 4：複合效率分（Efficiency Score）
Efficiency = ROCC_adj × PoP
```

**評分規則：**

| Efficiency Score | 分數 | 對應情境示例 |
|-----------------|------|------------|
| > 400 | 9 | ROCC=520%（adj）× PoP=0.80 = 416 |
| 250 – 400 | 7 | ROCC=330% × PoP=0.80 = 264 |
| 150 – 250 | 5 | ROCC=200% × PoP=0.80 = 160 |
| 80 – 150 | 3 | ROCC=150% × PoP=0.80 = 120 |
| < 80 | 1 | 最低保底（含所有有報價的合約） |

**Excel 展示欄位：**

| 欄位 | 範例 |
|------|------|
| Bid（目標履約價） | $4.90 |
| 目標履約價 | $98.00 |
| DTE | 32 |
| DTE 效率係數 | 1.10 |

### ⑩ 波動率偏斜（Skew ZScore）（4 分）— v5.0 全新維度

**研究依據：** CME CVOL Skew 教程、Koticha et al.《Sector Option Correlation Premiums》(SSRN 2024)、StrikeWatch 25-Delta Risk Reversal 研究

**為何重要：** 目前②維度（IVR）衡量的是整體波動率溢價，但 Sell Put 執行在 OTM 位置，OTM Put 因「波動率偏斜」額外高於 ATM——此額外溢價因市況不同而大幅波動。當偏斜處於歷史極端高位時，OTM Put 的溢價遠超 IVR 所顯示的水平。

**計算方式：**

```python
# Step 1：25-Delta 偏斜比（Skew Ratio）
# 25-Delta Put 的 IV 減去 ATM IV，再以 ATM IV 正規化
IV_25delta_put = options_chain.puts.loc[delta ≈ -0.25, 'impliedVolatility']
IV_atm         = options_chain.puts.loc[delta ≈ -0.50, 'impliedVolatility']  # ATM 近似

Skew_Ratio = (IV_25delta_put - IV_atm) / IV_atm × 100  # 單位：%

# Step 2：與 30 日滾動歷史均值和標準差比較
Skew_30d_avg = rolling_mean(Skew_Ratio, 30 days)
Skew_30d_std = rolling_std(Skew_Ratio, 30 days)

Skew_ZScore = (Skew_Ratio - Skew_30d_avg) / Skew_30d_std

# Step 3：若 30 日數據不足 → 保守給 2 分（中性）
```

**評分規則：**

| Skew ZScore | 分數 | 解讀 |
|------------|------|------|
| > +2.0 | 4 | OTM Put 被嚴重高估（相對歷史），溢價豐厚 |
| +1.0 至 +2.0 | 3 | OTM Put 相對高估，有額外偏斜溢價 |
| -1.0 至 +1.0 | 2 | 偏斜在歷史正常範圍 |
| < -1.0 | 1 | OTM Put 相對便宜，Sell Put 溢價不足 |
| 數據不足 | 2 | 保守中性 |

> **設計原則：** 偏斜的「高」是相對概念。Z 分數 > 2 代表今日 OTM Put 比過去 30 天 97.7% 的交易日更貴，是難得的賣方溢價窗口。

## 四、宏觀覆蓋——五體制系統（v5.0 升級）

**研究依據：** Kelly/VIX Hybrid（arXiv 2025）、Volmageddon 案例（2018 VIX+90%、2024 VIX=65.3）

**v4.0 問題：** 線性平滑過渡無法處理 VIX > 35 的極端情境；VIX 從 20→25 和 30→35 的影響被等同對待，但期權定價的非線性效應差異巨大。

### 五體制 VIX 分類表

| 體制 | VIX 範圍 | 情緒標籤 | Delta 調整 | 倉位縮放 | 操作建議 |
|------|---------|---------|----------|---------|---------|
| 1 | < 15 | 極度貪婪 | +0.05 | ×1.00 | 正常操作，適度進攻 |
| 2 | 15 – 20 | 正常中性 | 0 | ×1.00 | 標準操作 |
| 3 | 20 – 25 | 輕微焦慮 | −0.03 | ×0.85 | 降低 Delta，縮小倉位 |
| 4 | 25 – 35 | 恐慌 | −0.05 | ×0.70 | 大幅保守化，優先管理既有倉 |
| 5 | > 35 | 崩潰 | N/A | ×0.00 | **停止所有新倉**，全力管理現有部位 |

> **體制 5 說明（新增）：** VIX > 35 時，Gamma 風險急速擴大，IV Crush 反轉機率大幅上升。Iyer (SSRN 2025) 研究顯示：逆向波動率策略在 2018 年 Volmageddon 及 2024 年 12 月 VIX 飆升中，最大虧損超過 90%。此體制下不論評分多高，一律暫停新開倉。

### 修正後 Delta 計算

```python
base_delta_ranges = {
    'A': (0.25, 0.30),
    'B': (0.20, 0.25),
    'C': (0.15, 0.20),
}

vix_adjustments = {
    1: +0.05,   # VIX < 15
    2:  0.00,   # 15 ≤ VIX < 20
    3: -0.03,   # 20 ≤ VIX < 25
    4: -0.05,   # 25 ≤ VIX < 35
    5:  None,   # VIX ≥ 35：停止新倉
}

adj_lo = max(0.10, base_lo + vix_adj)
adj_hi = max(0.12, base_hi + vix_adj)
```

## 五、透明計算鏈

```
Step 1  子分加總
        原始總分 = ①+②+③+④+⑤+⑥+⑦+⑧+⑨+⑩

Step 2  基本面門檻檢查
        if ③基本面 < 10:
            調整後總分 = 原始總分 × 0.8
        else:
            調整後總分 = 原始總分

Step 3  等級判定
        A ≥ 80 ／ B 65–79 ／ C 50–64 ／ D < 50

Step 4  宏觀體制判定
        if VIX > 35: 停止新倉（體制5）
        else: 計算 Delta 調整 + 倉位縮放係數

Step 5  Delta 建議
        基礎 Delta（依等級）→ 套用 VIX 體制調整 → 修正後 Delta 範圍

Step 6  Beta 相關性評估
        計算加入此標的後的板塊集中度與 Beta 貢獻（影響⑤分數）

Step 7  板塊集中度警告
        if 板塊比例 > 40% AND 板塊 3 月動能為負: 🚫 高風險警告
        if 板塊比例 > 40% AND 板塊 3 月動能為正: ⚠️ 注意警告

Step 8  事件風險核查
        if ⑦ = 0（財報 < 7 天）: 標記 🚫，不論評分多高禁止新倉
```

## 六、退出框架（v5.0 精確量化版）

**研究依據：** Tastytrade Managing Winners 研究（SPY 2005-2025 測試結果）、Optopsy 回測框架

### 6.1 核心出場規則（50% 利潤關倉）

**Tastytrade 研究結論：** 在 SPY 進行 2005-2025 的 16-Delta 45 DTE 多年測試中：
- 持倉至到期：勝率最高，但 P/L per day 和 Sharpe 最低
- **50% 利潤平倉**：勝率略降 5-8%，但 P/L per day 顯著提升 23-31%，夏普比大幅改善
- 最優規則：**50% 利潤 OR 2× Credit 損失 → 取先到者關倉**

### 6.2 量化出場觸發條件

| 優先級 | 觸發條件 | 建議動作 | 理由 |
|--------|---------|---------|------|
| 🔴 P0 | **利潤 ≥ 50% 收取溢價** | 立即關倉（Buy to Close） | Tastytrade 研究最優出場點 |
| 🔴 P0 | **損失 ≥ 2× 收取溢價** | 強制評估：平倉或滾倉 | 損失控制上限 |
| 🔴 P0 | 財報日 < 7 天 | 強烈建議提前平倉 | 二元風險 |
| 🟠 P1 | **DTE < 21 天** | 評估提前關倉 | Gamma 激增期開始（Bondarenko 2024） |
| 🟠 P1 | 股價跌破履約價 10% | 評估滾倉（見 6.3） | 管理被行權風險 |
| 🟠 P1 | VIX 體制升至 4 或 5 | 縮減組合 Delta 暴露 | 系統性風險上升 |
| 🟡 P2 | 同板塊 ≥ 2 筆同時被行權 | 暫停板塊新倉 | 相關性集中爆發 |

### 6.3 滾倉規則（Roll Down & Out）

```
滾倉條件（三者同時滿足）：
  ① 基本面評分仍 ≥ 15 分（有長期持有價值）
  ② 距到期日 > 14 天（仍有時間溢價）
  ③ 新合約 Efficiency Score（ROCC × PoP） ≥ 80（不倒貼保費）

滾倉執行：
  Buy to Close（原有 Put）
  Sell to Open（更低履約價 × 更遠到期日，目標 30-45 DTE）
  淨溢價差 ≥ $0（不可倒付保費滾倉）

最大滾倉次數：同標的不超過 2 次（超過 → 公司趨勢確認向下，止損）
```

### 6.4 被行權後 Covered Call 流程

```
接股後：
  Step 1  重算基本面分數
          if 基本面 < 10: 考慮直接賣出，不做 Covered Call
          if 基本面 ≥ 15: 進入 Covered Call 策略

  Step 2  Covered Call 目標
          履約價 ≈ 持股成本（Put 履約價 − 收取權利金）
          Delta 目標：0.20 – 0.30

  Step 3  Exit 條件
          if 股價回升至持股成本: 評估是否了結
          if 持倉 > 90 天未回升: 重新評估是否繼續
```

### v5.0 動態警告邏輯

```python
# 計算板塊集中度
sector_pct = sector_count / total_watchlist

# 計算板塊 3 個月動能（相對 SPX）
sector_etf = {'半導體': 'SOXX', '科技消費': 'XLK', '電商/雲端': 'XLY', ...}
sector_3m_return = (sector_etf.price_today / sector_etf.price_90d_ago) - 1
spx_3m_return   = (SPX.price_today / SPX.price_90d_ago) - 1
sector_momentum = sector_3m_return - spx_3m_return  # 相對動能

# 發出警告
if sector_pct > 0.40:
    if sector_momentum > 0:
        warning = f"⚠️ {sector} {sector_count}/{total}（{sector_pct:.0%}）— 板塊強勢，需注意若動能反轉的系統風險"
    else:
        warning = f"🚫 {sector} {sector_count}/{total}（{sector_pct:.0%}）— 板塊動能為負，高度集中下系統性風險已顯現"
```

## 十一、操作警告清單（v5.0 版）

1. **VIX > 35 → 停止所有新倉**（Volmageddon 教訓：2018 逆波動策略最大虧損 90%+）
2. **財報 < 7 天 → 不執行**（二元風險無法量化）
3. **期權 Spread > 10% → 不執行**（滑點吞噬全部溢價）
4. **FCF ≤ 0 → 基本面 × 0.8**（被行權後接的是真實損失，非帳面虧損）
5. **IVR < 20% → 慎重**（溢價薄，Sell Put 效率不高）
6. **Skew ZScore < -1 → 注意**（OTM Put 相對便宜，賣方優勢不足）
7. **Delta ≠ 被行權機率**（Black-Scholes Delta 是 hedge ratio）
8. **PoP × ROCC > 理論值**（實際有保證金、手續費、滑點）
9. **PE 類型不一致不可比較**（同批次確認使用相同 PE 類型）
10. **50% 利潤就關倉**（Tastytrade 研究：超過 50% 後 Theta 收益 << Gamma 風險）

## 十二、Optopsy 回測驗證框架（v5.0 新增）

**研究依據：** [Optopsy GitHub (michaelchu/optopsy)](https://github.com/michaelchu/optopsy)、[Options Portfolio Backtester (lambdaclass)](https://github.com/lambdaclass/options_backtester)

### 回測目標（每季執行一次）

```python
# 使用 Optopsy 驗證各等級的實際表現
# 過濾條件對應模型的評分標準
import optopsy as op

results = op.backtest(
    data        = historical_options_data,
    strategy    = op.short_put,
    filters     = {
        'delta'        : (-0.25, -0.15),   # 目標 Delta 範圍
        'dte'          : (21, 45),          # DTE 窗口
        'entry_spread' : (0, 0.05),         # Spread < 5%（流動性）
        'min_oi'       : 1000,              # 最低 OI
    },
    exit_rules  = {
        'profit_target' : 0.50,  # 50% 利潤出場
        'stop_loss'     : 2.00,  # 2× 損失出場
        'dte_exit'      : 21,    # DTE = 21 強制評估
    }
)
```

### 驗證標準（各等級預期目標）

| 等級 | 預期勝率 | 預期被行權率 | P/L per day |
|------|---------|------------|------------|
| A (≥ 80) | > 82% | < 12% | 最高 |
| B (65-79) | > 72% | < 22% | 中高 |
| C (50-64) | > 60% | < 32% | 中低 |

> 若實際資料與預期標準偏離超過 10 個百分點，代表評分閾值需重新校正。建議每季至少積累 50 筆歷史交易記錄後再進行回測驗證。
